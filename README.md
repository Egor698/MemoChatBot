# 🤖 Описание агента

Стандартные LLM не способны качественно хранить историю общения при большом контексте, а также увеличивается количество токенов.

👉 Результат: агент становится забывчивым и дорогим.

Решение

Мой агент решает проблему долгосрочной памяти через гибридный подход:

🧑 Профиль пользователя — хранит ключевую информацию о пользователе.

📜 Буфер диалога — хранит последние n сообщений.

🔁 Суммаризация — когда буфер заполнен, агент автоматически:
* пересказывает накопленный диалог в краткое резюме
* обновляет профиль пользователя на основе нового контекста
* очищает буфер (старые сообщения удаляются, суть сохранена)

🧠 Результирующий промпт = краткое резюме + последние сообщения (один диалог всегда) + текущий запрос

✅ Агент помнит долгосрочный контекст, экономя токены и сохраняя смысл.

# Использование как **Telegram Bot**
